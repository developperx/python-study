# 作問ガイド

このアプリの問題は `tools/define_questions.py` に Python で定義し、
`tools/build_questions.py` で `data/chNN.json` を生成します。

## 大原則（法的・品質）

1. **公式試験問題や教材『Python実践レシピ』の文章を転載しない。** すべて自分の言葉でオリジナルに作問する。
2. 出題は **Python 本体・標準ライブラリ** の範囲に限る（サードパーティ製ライブラリは試験範囲外。参考として触れる場合はその旨を明記）。
3. 1問につき**正解は明確に1つ（または複数選択なら複数）**で、誤答選択肢も紛らわしすぎない妥当なものにする。
4. **解説は必ず付ける。** なぜ正解か／他がなぜ誤りかが分かるように書く。

## 関数シグネチャ

```python
q(chapter, topic, difficulty, question, choices, answer,
  explanation, reference="", code=None, type="single", verify=False, rat=None)
```

| 引数 | 説明 |
|------|------|
| `chapter` | 章番号（1,2,3,4,5,6,8,9,10,11,13,14,16,17,18 のいずれか） |
| `topic` | 小分類のラベル（例: "内包表記"） |
| `difficulty` | 1=易 / 2=標準 / 3=難 |
| `question` | 問題文 |
| `choices` | 選択肢のリスト（2件以上） |
| `answer` | 正解の **0始まりインデックス**（複数選択は `[0, 2]` のようにリスト） |
| `explanation` | 総合解説（なぜ正解かの全体説明） |
| `reference` | 学習リファレンス（任意。公式ドキュメント名など。書籍の引用はしない） |
| `code` | 表示するコード（任意） |
| `type` | `"single"` / `"multiple"` |
| `verify` | `True` にすると `code` を実行し stdout が正解選択肢と一致するか自動検証 |
| `rat` | **選択肢ごとの正誤理由**のリスト（`choices` と同じ長さ）。各選択肢が「なぜ正解／不正解か」を書く |

`id` は `chNN-XXX` の形式で自動採番される。

### 選択肢ごとの理由（`rat`）

誤答を含む全選択肢に個別の理由を付けるため、`rat` に `choices` と同じ件数の理由を渡す。
先頭の「**正解。**／**不正解。**」は build 側で**自動付与**されるので、理由本文だけを書けばよい。

```python
q(3, "数値", 1, "次のコードの出力はどれか。", ["3", "2", "2.5", "2.0"], 0,
  "`//` は切り捨て除算。7//2=3。", "算術演算 --- //",
  code="print(7 // 2)", verify=True,
  rat=["7//2 は商の整数部 3。",         # → 「正解。7//2 は…」と表示される
       "2 ではなく 3。",                # → 「不正解。2 ではなく 3。」
       "// は小数を返さない。",
       "整数同士の // は float にならない。"])
```

`rat` を省略すると、正解選択肢は「正解。」、それ以外は「不正解。」とだけ表示される（理由を書くことを推奨）。
`validate_data.py` は `rationales` の件数が `choices` と一致するか検証する。

## verify=True の使い方（出力予測問題）

「次のコードの出力は？」という問題は `verify=True` を付けると、ビルド時に
**実際に Python で `code` を実行し、その標準出力が正解選択肢の文字列と一致するか**を検証する。
これにより正解ミスを機械的に検出できる。

注意点:
- 出力と**完全一致**（前後の空白は strip）する必要がある。
- 複数行出力は選択肢に**本物の改行**（`"a\nb"`）を入れる。`"a\\nb"`（バックスラッシュ+n）にしない。
- `print(repr(...))` を使うと、クォートを含む期待値を厳密に表せる。
- 環境非依存にする（時刻・乱数・OS依存のパスなどは `verify=False` にするか、出力に依存しない形にする）。

```python
q(3, "内包表記", 1,
  "次のコードの出力として正しいものはどれか。",
  ["[0, 1, 4, 9]", "[1, 2, 3]", "[0, 1, 2, 3]", "[1, 4, 9]"],
  0,
  "リスト内包表記。range(4)=0,1,2,3 の各2乗で [0, 1, 4, 9]。",
  "リスト内包表記",
  code="print([x**2 for x in range(4)])", verify=True)
```

## ビルドと検証

```bash
python3 tools/build_questions.py   # data/ を再生成。verify=True を実行検証
python3 tools/validate_data.py     # id一意・answer範囲・必須項目・模試比率を検証
```

どちらもエラーが無ければOK。CI（GitHub Actions）でも同じ検証が走る。

## 模試の出題比率を変えるとき

模試の章別出題数は **2か所**に定義されており、一致させる必要がある:

- `js/data.js` の `EXAM.distribution`
- `tools/validate_data.py` の `DISTRIBUTION`
